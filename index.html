LLM Effort Visualizer — Dataset Findings

53 prompts across 5 difficulty tiers, run through Mistral 7B via Ollama with max_tokens=256. The goal: can surface prompt features predict inference effort? The answer is illuminating.

Finding 1 — The Predictor
Surface prompt features cannot predict generation time
R² = −1.51

A Ridge regression over 8 prompt features (word count, logic words, constraint words, question marks, etc.) fitted to generation time achieves a cross-validated R² of −1.51 — substantially worse than simply predicting the mean every time. The model learned nothing useful from the prompt surface.

Why? The dominant driver of generation time is output token count, which in this run is almost entirely determined by the 256-token cap — not by prompt complexity. Once you control for output tokens (r²=0.84 between output tokens and gen_ms), the residual variance isn't explained by anything visible in the prompt text.

Implication: To fix the predictor, you need either (a) a higher or no token cap so output length varies freely with problem complexity, or (b) a different target variable. Tokens/sec is a better candidate — see Finding 2.
Finding 2 — The Counterintuitive Signal
Harder prompts make the model run faster

Tokens per second — a real hardware-level inference signal — trends upward with difficulty. Trivial prompts average 4.24 t/s; hard prompts average 5.61 t/s. This is counterintuitive and reproducible across the dataset.

Hypothesis: Hard prompts (math, CS theory, well-defined technical questions) fall into well-worn token sequences the model has seen many times during training. The KV-cache warms up faster, attention patterns are sparser, and the model produces fluent output quickly. Creative and narrative prompts (the slowest: "Write a short story" at 3.12 t/s) require the model to sample in a more diffuse, high-entropy space — many plausible continuations, slower selection.
Finding 3 — The Token Cap Confound
Generation time clusters into two regimes separated by the cap

The dataset splits cleanly into prompts answered in <256 tokens (short, discrete answers) and those that hit the cap (open-ended questions the model could answer indefinitely). Within the capped group, generation time ranges from 41–82 seconds — driven almost entirely by t/s variance, not by "more thinking."

Fix: Re-run with --max-tokens 2048 and open-ended prompts that actually differ in answer depth. The variance you want (complex question → more tokens → longer time) will only appear when the cap isn't the binding constraint.
Finding 4 — Cognitive Churn
Mistral barely backtracks — churn is nearly zero everywhere

Only 2 of 53 prompts produced any backtracking language in the output. Both are deceptive-simple prompts (the hotel room paradox and the quantum entanglement explanation). The churn score — pivot phrases per 100 output tokens — peaks at 0.5 across the entire dataset.

Why so flat? Mistral 7B at 256 tokens rarely has room to backtrack — it answers and stops. Churn as a signal requires a model that (a) has a thinking trace separate from output, or (b) is given enough tokens to reconsider. Claude with extended thinking showed meaningful churn variance across difficulty. For Ollama, churn would need a much higher token budget and/or a model that's more self-critical.

What to do next

Three concrete changes that would make the predictor actually work:

1. Remove the token cap. Re-run with --max-tokens 2048 or higher. Let output length be free — then complexity genuinely drives time.

2. Use t/s as the target, not gen_ms. Tokens per second already shows the counterintuitive difficulty signal. Train the regressor on t/s and you're predicting something real about inference dynamics rather than wall clock time dominated by output length.

3. Switch to a reasoning model. Pull deepseek-r1:7b or qwen3:8b from Ollama — both expose thinking traces in <think> tags. Then you get reasoning tokens as a real ground-truth signal, the same way the Anthropic version used extended thinking. That's the signal the predictor actually needs.

The core finding stands: prompt surface features don't straightforwardly predict inference effort, and that's interesting. The question of what does predict it — and whether any pre-inference classifier can beat the mean — is still open.
